En el desarrollo del proyecto de la aplicación de streaming en vivo, se ha trabajado en la planificación y diseño de la arquitectura del sistema utilizando el patrón Modelo-Vista-Controlador (MVC). Se definieron las funcionalidades principales, como la creación y gestión de transmisiones en vivo, la interacción en tiempo real a través de chats y reacciones, y la personalización de la experiencia del usuario.

Para el **backend**, se utilizó **Java** con el framework **Spring Boot**, que permite una integración sólida y segura de la lógica del negocio, la gestión de usuarios y la administración de datos. El **frontend** fue desarrollado con **HTML, CSS y JavaScript**, brindando una interfaz de usuario intuitiva y dinámica. Para las plantillas y vistas, se utilizó **Thymeleaf**, que facilita la integración y comunicación directa con el backend de Java, permitiendo la actualización dinámica de las vistas basadas en la lógica de la aplicación.

Se estableció un backlog del producto detallado, priorizando las funcionalidades esenciales para la primera versión del sistema. Se desarrollaron sprints con objetivos claros, como la implementación de la funcionalidad de edición de video y herramientas básicas de moderación. También se abordaron aspectos como la escalabilidad, seguridad y usabilidad del sistema, asegurando una experiencia fluida y segura para los usuarios.

Adicionalmente, se crearon diagramas de casos de uso, clases y actividades para representar las interacciones y flujos del sistema. Se definieron vistas lógicas y de implementación para la organización del código, y un diagrama de despliegue para visualizar cómo los diferentes componentes del sistema se alojarán en el entorno de producción.
